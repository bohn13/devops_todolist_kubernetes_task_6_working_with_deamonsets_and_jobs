Даний репозиторій містить маніфести для розгортання додатку `todoapp` разом із допоміжними компонентами для моніторингу та автоматизації в Kubernetes.

## 1. Розгортання (Deployment)

Перш ніж розгортати `DaemonSet` та `CronJob`, переконайтеся, що основний додаток та його сервіс запущені в неймспейсі `mateapp`.

```powershell
# Застосування основних маніфестів
kubectl apply -f deployment.yml
kubectl apply -f service.yml  # Переконайтеся, що назва сервісу todoapp-service
```

### Розгортання нових компонентів:

```powershell
# Застосування DaemonSet
kubectl apply -f daemonset.yml

# Застосування CronJob
kubectl apply -f cronjob.yml
```

---

## 2. Пояснення конфігурації

### DaemonSet (`daemonset.yml`)

* **Образ:** `radial/busyboxplus:curl` (містить необхідну утиліту `curl`).
* **Логіка:** Запускає по одному поду на кожній ноді кластера. Кожні 5 секунд виконується цикл  який перевіряє доступність `todoapp` через внутрішню мережу кластера (ClusterIP).
* **Ресурси:** Встановлено ліміти та реквести для запобігання неконтрольованому споживанню ресурсів системними подами.

### CronJob (`cronjob.yml`)

* **Розклад:** `*/4 * * * *` (кожні 4 хвилини).
* **Concurrency Policy:** `Allow` — дозволяє запуск нового завдання, навіть якщо попереднє ще не завершилося.
* **Історія:** Зберігає 10 успішних та 5 невдалих запусків для зручного аудиту.
* **Логіка:** Одноразовий виклик `/api/health` для перевірки стану додатка.

---

## 3. Валідація рішення (Перевірка логів)

Щоб переконатися, що все налаштовано правильно, виконайте наступні перевірки:

### Перевірка DaemonSet:

Знайдіть поди пінгера та перегляньте їхні логи. Ви маєте побачити HTTP відповіді кожні 5 секунд.

```powershell
# Отримати імена подів DaemonSet
kubectl get pods -n mateapp -l app=todoapp

# Перегляд логів (замініть <pod-name>)
kubectl logs <pod-name> -n mateapp
```

### Перевірка CronJob:

CronJob створює окремі поди для кожного запуску. Дочекайтеся виконання (мінімум 4 хвилини) та перевірте результат.

```powershell
# Отримати список виконаних завдань (Jobs)
kubectl get jobs -n mateapp

# Отримати поди, створені CronJob
kubectl get pods -n mateapp | Select-String "todoapp-cron"

# Перегляд логів виконаного завдання
kubectl logs <pod-name-from-cronjob> -n mateapp
```
